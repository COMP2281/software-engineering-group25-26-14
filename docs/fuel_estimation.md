## Highly Accurate Fuel Estimation Module

The fuel estimation module resides in `analytics_engine/fuel_estimation/` and is responsible for calculating precise vehicle fuel consumption efficiency.

This module takes a `ProcessedTrip` (generated by the Preprocessing Pipeline) and a `VehicleProfile` (calibrated by the Digital Twin Builder) to compute a highly accurate "Liters per 100 Kilometers" (L/100km) metric.

### Architecture and Logic

The core of the module is in `estimator.py`, which exposes the `FuelEstimator` class. It heavily relies on vectorized Pandas operations to process thousands of OBD-II telematics records in milliseconds.

#### The Mathematical Approach

To avoid the inaccuracies of generic engine load mapping, this model directly calculates the mass of air entering the engine and pairs it with the exact air-to-fuel ratio dictated by the vehicle's fuel type. 

1. **Interval Calculation (`time_delta_s`)**: 
   - The gap between each sequential row is calculated in seconds using the `Timestamp` column.
   - **Resiliency handling:** Any anomalies in the time delta (negative jumps or gaps exceeding 10 minutes that shouldn't exist inside a cohesive trip segment) are clipped to a safe 1-second interval to avoid inflating distance/fuel artificially.
2. **Total Distance (km)**:
   - For every interval: \`Distance = (Speed / 3600.0) * time_delta_s\`
   - These are summed to calculate the trip's exact total distance.
3. **Fuel Mass (grams)**:
   - Using the exact `stoichiometric_afr` obtained from the `VehicleProfile`, the module calculates how many grams of fuel were burnt per second.
   - For every interval: \`Fuel Mass = (MAF / AFR) * time_delta_s\`
4. **Fuel Volume (Liters)**:
   - The module maps the `fuel_type` from the Digital Twin to a known specific density constant (e.g., Petrol ~745 g/L, Diesel ~832 g/L). 
   - \`Total Liters = Total Fuel Mass / Fuel Density\`
5. **Efficiency (L/100km)**:
   - The final metric is produced by normalizing the burnt liters over the distance traveled.
   - \`(Total Liters / Total Distance) * 100.0\`

### Resiliency and Graceful Fallbacks

The `FuelEstimator` is designed to be highly fault-tolerant against imperfect real-world dataset captures and API timeouts:
- Missing required columns (`MAF`, `Speed`, `Timestamp`) exit the calculation gracefully with `NaN`.
- Broken or absent `stoichiometric_afr` properties in the profile automatically fall back to `14.7` (Standard Petrol).
- Unrecognized or absent fuel types default to standard Petrol densities. (Electric vehicles return `NaN` since you cannot natively calculate L/100km without deep kWh conversion parameters).

### API/Programmatic Usage

The calculation is entirely stateless and exposes a clean `@staticmethod`. It requires the `VehicleProfile` and a `ProcessedTrip` object.

```python
from analytics_engine.fuel_estimation.estimator import FuelEstimator

# Assume `profile` is a VehicleProfile from VehicleProfileBuilder
# Assume `trip` is a ProcessedTrip from PreprocessingPipeline
estimator = FuelEstimator()

# Calculate fuel efficiency
fuel_efficiency = estimator.calculate_trip_fuel_consumption(profile, trip)

if not pd.isna(fuel_efficiency):
    print(f"Trip Efficiency: {fuel_efficiency:.2f} L/100km")
else:
    print("Fuel estimation could not be calculated for this trip.")
```
